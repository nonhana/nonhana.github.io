   事后我问了问 GPT，接下来贴一下它的回答：

   > "怪异盒模型"（Quirks Mode Box Model）是一个在早期的浏览器版本中使用的 CSS 盒模型。在这个模式下，元素的 `width` 和 `height` 属性不仅包括内容的尺寸，还包括了内边距（padding）和边框（border）。这与现代浏览器中默认使用的标准盒模型（W3C Box Model）是不同的。在标准盒模型中， `width` 和 `height` 仅仅定义了内容区域的尺寸，而内边距和边框则是额外添加的。
   >
   > 怪异盒模型主要是为了向后兼容那些在早期浏览器规范成型之前就已经存在的网站。现代浏览器通常允许开发者通过设置文档类型（DOCTYPE）来选择使用哪种盒模型。
   >
   > 如果你在开发过程中遇到了怪异盒模型，你可以通过设置 CSS 的 `box-sizing` 属性为 `border-box` 或 `content-box` 来明确指定使用哪种盒模型。
   >
   > - `border-box` ：这会使元素采用怪异盒模型的行为，其中 `width` 和 `height` 包括内容、内边距和边框。
   > - `content-box` ：这会使元素采用标准盒模型的行为，其中 `width` 和 `height` 仅包括内容。

10. **讲讲 CSS 中可继承的属性有哪些。** 一开始题目都没听懂，问是不是 JS 的，结果跟我说是 CSS 的。我又一次完全没了解过的问题出现了，直接说不会。

    问问 GPT 吧：

    > 在 CSS 中，有些属性是可继承的，而有些则不是。可继承的属性意味着子元素会继承其父元素的这些属性值，除非子元素自己有明确的设置。以下是一些常见的可继承的 CSS 属性：
    >
    > **文本相关属性**
    >
    > 1.  `color`：文本颜色
    > 2.  `font`：字体设置（这包括 `font-family` 、 `font-size` 、 `font-weight` 等）
    > 3.  `text-align`：文本对齐方式
    > 4.  `text-indent`：文本缩进
    > 5.  `line-height`：行高
    > 6.  `letter-spacing`：字符间距
    > 7.  `text-transform`：文本转换（如大写、小写等）
    > 8.  `white-space`：空白字符的处理方式
    > 9.  `word-spacing`：单词间距
    >
    > **列表相关属性**
    >
    > 1.  `list-style`：列表样式（这包括 `list-style-type` 、 `list-style-position` 等）
    >
    > **表格相关属性**
    >
    > 1.  `border-collapse`：边框合并设置
    > 2.  `border-spacing`：边框间距
    > 3.  `caption-side`：表格标题位置
    > 4.  `empty-cells`：空单元格的显示方式
    >
    > **其他**
    >
    > 1.  `visibility`：可见性（注意： `hidden` 值不会被继承，但子元素会表现得像继承了）
    > 2.  `cursor`：光标样式
    > 3.  `direction`：文本方向
    >
    > 这些只是一些常见的可继承属性，不是一个完整的列表。具体哪些属性是可继承的，还取决于浏览器的实现和 CSS 规范的版本。

11. **CSS 中有三个属性可以让某个 DOM 隐藏，分别是： `display:none` 、 `visable:hidden` 、 `opacity:0` ，讲讲这三种方式彼此之间有什么区别？** 由于第二种用的不多，答得一般，没太分析出彼此的不同。

    > 在 CSS 中， `display: none` 、 `visibility: hidden` 和 `opacity: 0` 都可以用于隐藏 DOM 元素，但它们之间有一些重要的区别：
    >
    > **`display: none`**
    >
    > 1. **渲染效果**：元素不会被渲染，即完全消失，不占用任何空间。
    > 2. **文档流**：元素会从文档流中移除，就像它从未存在一样。
    > 3. **可访问性**：对于屏幕阅读器等辅助技术，元素及其子元素也会被隐藏。
    > 4. **性能**：如果频繁地切换该属性，可能会导致页面重新布局，影响性能。
    > 5. **事件响应**：元素不会响应任何用户事件（如点击、悬停等）。
    >
    > **`visibility: hidden`**
    >
    > 1. **渲染效果**：元素会被隐藏，但仍然占用空间。
    > 2. **文档流**：元素仍然存在于文档流中。
    > 3. **可访问性**：对于屏幕阅读器，元素（通常）也会被隐藏。
    > 4. **性能**：性能开销相对较小，因为不需要重新布局。
    > 5. **事件响应**：元素不会响应任何用户事件（如点击、悬停等）。
    >
    > **`opacity: 0`**
    >
    > 1. **渲染效果**：元素会变得完全透明，但仍然可见（例如，你仍然可以看到其背后的内容）。
    > 2. **文档流**：元素仍然存在于文档流中。
    > 3. **可访问性**：元素对于屏幕阅读器是可见的。
    > 4. **性能**：通常不会导致重新布局，但可能会影响绘制性能。
    > 5. **事件响应**：元素仍然会响应用户事件（如点击、悬停等）。
    >
    > 这些属性各有用途，选择哪一个取决于你想达到什么效果。例如，如果你想让元素消失但不影响布局，`visibility: hidden` 是一个好选择。如果你想让元素完全消失并从文档流中移除，`display: none` 更合适。如果你只是想让元素暂时不可见但仍能交互，`opacity: 0` 是一个好选项。
